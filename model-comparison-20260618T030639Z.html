Scoring methodology

Section matching

Greedy best-match pairing by layout_type, then fields, column headers (set F1), and Jinja variables — not index position.

Overall weights

Metadata 5% · layout multiset 15% · section match 30% · fields F1 25% · variables F1 15% · column headers 5% · section count 5%.

Fairness rules

numbered_list ≡ ordered_list. grid_position ignored (post-processing). Content scored via Jinja variable overlap. temperature=0 for gpt-4.1; gpt-5.5 uses model default. Textract cached per PDF.
